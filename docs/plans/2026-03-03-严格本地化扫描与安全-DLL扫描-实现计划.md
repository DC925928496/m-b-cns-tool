# 严格本地化扫描与安全 DLL 扫描 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 严格按《骑马与砍杀2》本地化规范收敛扫描范围：只翻译 `std_module_strings_xml.xml`；从 XML/JSON/DLL 中仅收集 `{=id}` 引用用于补全语言文件；DLL 扫描仅扫描 `SubModule.xml` 声明的主 DLL，且不再生成 `runtime_localization.json`。

**Architecture:** `ModTextExtractor` 负责构建“规范语言文件”的唯一真源（`ModuleData/Languages/std_module_strings_xml.xml`），并从 Mod 数据与主 DLL 中收集 `{=id}` 引用来补全缺失条目。翻译阶段仅对语言文件的 `<string id=... text=...>` 进行翻译与写回，其它文件不再参与“句子级”翻译与补丁生成。

**Tech Stack:** .NET 8、xUnit、Mono.Cecil、XDocument、System.Text.Json

### Task 3：实现严格扫描（让测试变绿）

**Files:**
- Modify: `MbCnsTool.Core/Extraction/ModTextExtractor.cs`
- Modify: `MbCnsTool.Core/Extraction/DllStringScanner.cs`

**Step 1: Minimal implementation**
- `ModTextExtractor`：
  - 只加载/生成 `ModuleData/Languages/std_module_strings_xml.xml` 作为唯一输出文档；
  - 扫描其它 XML/JSON 仅收集 `{=id}` 引用（不产出 `TextUnit`）；
  - 扫描主 DLL（见下）收集 `{=id}` 引用；
  - 将缺失的 id 以 `<string id="..." text="默认文本" />` 形式补全到语言文件；
  - 最终仅为语言文件 `<string>` 生成 `TextUnit`。
- `DllStringScanner`：
  - 从 `SubModule.xml` 解析 `<DLLName value="...">`；
  - 仅扫描被声明的 DLL 文件；
  - 仅提取 `"{=id}默认文本"` 这种高置信本地化字符串作为“引用”。

**Step 2: Run tests to verify pass**
- Run: `dotnet test MbCnsTool.Tests/MbCnsTool.Tests.csproj -c Release`
- Expected: PASS
